# База знаний — репозиторий управляемой базы знаний

## Назначение репозитория

**knowledge-base** — это централизованная база знаний, в которой контент создается и поддерживается автоматически с помощью ИИ-агента, а структура остается строгой, предсказуемой и масштабируемой.

Репозиторий предназначен для:

- **накопления экспертных знаний**, инструкций, методик, справочников;
- **превращения разрозненных файлов** (PDF, DOCX, Markdown, таблиц и т. д.) в структурированные статьи;
- **поддержки единого стандарта** оформления и навигации;
- **дальнейшего использования базы знаний ИИ-агентами** (чат-ботами, Copilot, RAG, поиском).

### Ключевой принцип

**одна статья = одна папка = один файл README.md**

## Структура репозитория

```
knowledge-base/
├── README.md              # Входная точка, описание и правила
├── inbox/                 # Сырые загрузки пользователя
│   └── README.md
├── kb/
│   └── ru/
│       ├── INDEX.md       # Центральный индекс статей (RU)
│       └── <topic>/
│           ├── README.md  # Лист темы (оглавление)
│           └── <article>/
│               └── README.md  # Статья
├── _templates/
│   └── article.md         # Шаблон статьи
├── prompts/
│   └── batch_upload_prompt.md  # Промт для обработки одной партии файлов
```

## Роли папок

### README.md (корень)

- объясняет, что это за репозиторий;
- описывает правила добавления контента;
- указывает, где находится индекс и как читать базу знаний.

### inbox/

**Единственная точка входа для новых материалов**

Сюда пользователь загружает:
- файлы в формате Markdown
- документы (PDF, DOCX)
- архивы
- таблицы
- заметки

**Важно:**
- Содержимое `inbox/` считается сырым и временным
- Пользователь не сортирует файлы вручную — это делает агент

### kb/ru/

**Каноничная база знаний (русский язык)**

- Здесь публикуются только готовые статьи
- Никакие черновики или «сырье» сюда не попадают
- `kb/ru/INDEX.md` — центральный реестр всех статей, автоматически обновляется агентом

#### Темы и статьи

- Каждая тема (`<topic>`) — отдельная папка
- Внутри темы:
  - `README.md` — лист темы (оглавление, описание, ссылки)
  - папки со статьями

### _templates/

- Хранит стандарты оформления
- `article.md` — эталон структуры статьи
- Агент использует его при создании новых материалов

### prompts/

- Хранит рабочие промты для ИИ-агента
- `batch_upload_prompt.md` — основной промт, который запускается каждый раз при обработке новых файлов

## Как работает агент

### Обработка одной партии загрузок

Когда в `inbox/` появляются новые файлы, агент выполняет строго определённый алгоритм:

1. **Сканирует inbox/**
   - определяет типы файлов
   - извлекает текст (если возможно)

2. **Классифицирует контент**
   - определяет тему
   - решает, один это материал или несколько статей

3. **Создаёт структуру**
   - при необходимости создает новую тему
   - создает папку со статьями

4. **Формирует статью**
   - копирует структуру из `_templates/article.md`
   - заполняет:
     - заголовок
     - краткое содержание
     - контекст применения
     - основные пункты
     - примеры
     - связи с другими статьями
   - Если данных недостаточно, отображается пометка `[[TBD]]`

5. **Обновляет индексы**
   - `kb/ru/INDEX.md`
   - лист темы (`kb/ru/<topic>/README.md`)

6. **Дедупликация**
   - проверяет, нет ли уже такой же статьи
   - не создает дубликаты
   - при совпадении — обновляет существующую статью

7. **Очистка inbox**
   - исходные файлы считаются обработанными
   - не используются напрямую для чтения базы знаний

## Стандарты контента

### Каждая статья:

- автономна (читается без контекста)
- не ссылается на «внутренние мысли»
- выдержана в едином стиле
- легко индексируется и анализируется искусственным интеллектом

### Каждая тема:

- логически цельная
- не содержит текста статьи — только навигацию

## Как использовать

### Добавление нового контента

1. Поместите файлы в папку `inbox/`
2. Запустите ИИ-агента для обработки
3. Агент автоматически:
   - проанализирует контент
   - создаст структурированные статьи
   - обновит индексы
   - очистит `inbox/`

### Навигация по базе знаний

- Начните с [центрального индекса](/kb/ru/INDEX.md)
- Перейдите к интересующей теме
- Читайте статьи в каждой теме

## Преимущества подхода

### 1. Масштабируемость
Можно загрузить 1000 файлов — структура останется читаемой.

### 2. Контроль качества
Никакого хаоса в «README_FINAL_v3.md».

### 3. Готовность к использованию ИИ
Репозиторий сразу готов к:
- RAG-системам
- GitHub Copilot
- чат-ботам
- внутреннему поиску

### 4. Пригодность для человека и машин
База знаний удобна и для людей, и для моделей.

## Коротко

Об этом репозитории:
- превращает загрузки в структурированную базу знаний
- применяет строгий стандарт (1 статья = 1 README)
- освобождает человека от ручной сортировки
- готов к автоматизации, масштабированию и использованию искусственного интеллекта
