# Структурированные данные подшипников

Описание структурированных данных, CSV-файлов и справочников по подшипникам, используемых в базе знаний.

## Контекст применения

- Понимание структуры данных номенклатуры подшипников
- Правила обновления справочников
- Трассировка источников данных
- Валидация и проверка данных
- Интеграция с внешними системами

## Основные наборы данных

### 1. Номенклатура производителей (nomenclature.csv)

**Расположение**: `kb/ru/bearings/datasets/nomenclature.csv`

**Назначение**: Кросс-справочник соответствий ГОСТ ↔ международные обозначения

**Характеристики**:
- Записей: 82,951
- Производителей: 85
- Формат: CSV (gost_designation, analog, manufacturer, source_url, source_file)

**Структура**:
```csv
gost_designation,analog,manufacturer,source_url,source_file
6305,6305,SKF,https://aprom.by/cgi-bin/nomenclature.pl?brand=SKF,SKF.md
6305-2RS,6305-2RSH,SKF,https://aprom.by/cgi-bin/nomenclature.pl?brand=SKF,SKF.md
```

**Источники данных**:
- inbox/processed/2026-02/SKF.md (10,000 позиций)
- inbox/processed/2026-02/FAG.md (5,075 позиций)
- inbox/processed/2026-02/NSK.md
- inbox/processed/2026-02/TIMKEN.md
- И еще 81 файл производителей

**Обновление**: При поступлении новых каталогов производителей в inbox/

**Валидация**:
- Уникальность комбинации (gost_designation, manufacturer)
- Наличие source_file в inbox/processed/
- Корректность URL источника

### 2. Каталог подшипников (catalog.csv)

**Расположение**: `kb/ru/bearings/datasets/catalog.csv`

**Назначение**: Основные характеристики типоразмеров подшипников

**Характеристики**:
- Записей: 73
- Серии: 62xx, 63xx, 7xxx, NU, NJ, 30xxx
- Формат: CSV

**Структура**:
```csv
designation,type,d_mm,D_mm,B_mm,dynamic_C_kN,static_C0_kN,rpm_limit,mass_kg
6305,ball_deep_groove,25,62,17,22.5,11.2,10000,0.13
```

**Источники**:
- Каталоги производителей (SKF, FAG, NTN)
- Стандарт ISO 15:2017 (габаритные размеры)
- Стандарт ISO 281:2007 (грузоподъёмность)

**Обновление**: При добавлении новых серий или типоразмеров

### 3. Справочник производителей (manufacturers.csv)

**Расположение**: `kb/ru/bearings/datasets/manufacturers.csv`

**Назначение**: Информация о производителях подшипников

**Характеристики**:
- Записей: 86
- Формат: CSV

**Структура**:
```csv
manufacturer,country,segment,features
SKF,Sweden,Premium,Полный каталог шариковых и роликовых
FAG,Germany,Premium,Специализация на прецизионных
NSK,Japan,Premium,Широкий ассортимент
```

**Источники**:
- inbox/processed/2026-02/*.md (85 файлов)
- Официальные каталоги производителей

**Обновление**: При добавлении нового производителя в номенклатуру

### 4. Таблица эквивалентов (equivalents.csv)

**Расположение**: `kb/ru/bearings/datasets/equivalents.csv`

**Назначение**: Соответствие обозначений между основными производителями

**Характеристики**:
- Записей: 89
- Формат: CSV

**Структура**:
```csv
base,skf,fag,ntn,nsk,gost,notes
6305,6305,6305,6305,6305,180305,Точное соответствие
6305-2RS,6305-2RSH,6305.2RSR,6305ZZ,6305DDU,180305Л,Отличия в уплотнениях
```

**Источники**:
- Кросс-справочники производителей
- nomenclature.csv (агрегация)

**Обновление**: При обнаружении новых соответствий

## Трассировка источников

### Пайплайн обработки данных

```
inbox/                          # Входящие данные
  ├── *.md                      # Каталоги производителей
  ├── *.xlsx                    # Прайс-листы
  └── *.pdf                     # Стандарты
          ↓
[Дедупликация]                  # Проверка sha256
  _meta/dedup_index.json
          ↓
[Нормализация]                  # Преобразование в CSV
          ↓
kb/ru/bearings/datasets/        # Структурированные данные
  ├── nomenclature.csv
  ├── catalog.csv
  ├── manufacturers.csv
  └── equivalents.csv
          ↓
[Генерация карточек]            # Создание README.md
  scripts/generate_bearing_cards.py
          ↓
kb/ru/bearings/cards/           # Карточки подшипников
  ├── 6305-2rs/README.md
  └── ...
          ↓
[Обновление индексов]           # Связывание
  kb/ru/bearings/INDEX.md
  kb/ru/podshipniki/INDEX.md
```

### Поле source в метаданных

Каждый файл данных содержит информацию об источнике:

**nomenclature.csv**:
- Колонка `source_file` → файл в inbox/processed/YYYY-MM/
- Колонка `source_url` → URL каталога производителя

**catalog.csv**:
- Комментарий в заголовке CSV → каталоги производителей

**manufacturers.csv**:
- Генерируется из списка файлов inbox/processed/

## Правила обновления данных

### Добавление новой номенклатуры

1. Положить файл производителя в `inbox/`
2. Запустить дедупликацию (sha256)
3. Нормализовать в формат nomenclature.csv
4. Обновить manufacturers.csv
5. Переместить в `inbox/processed/YYYY-MM/`
6. Обновить этот паспорт данных

### Добавление нового типоразмера

1. Найти данные в каталоге производителя
2. Проверить соответствие ISO 15 (габариты)
3. Добавить строку в catalog.csv
4. Сгенерировать карточку: `python scripts/generate_bearing_cards.py`
5. Обновить INDEX.md

### Обновление эквивалентов

1. Проверить в nomenclature.csv наличие соответствий
2. Добавить в equivalents.csv с указанием различий
3. Обновить карточки подшипников (секция "Эквиваленты")

## Валидация данных

### Автоматические проверки

```bash
# Проверка структуры CSV
python scripts/validate_csv_datasets.py

# Проверка уникальности ID
python scripts/kb_quality_gate.py

# Проверка ссылок
python tests/check_kb_links.py
```

### Ручные проверки

- Соответствие габаритов ISO 15
- Корректность грузоподъёмности (ISO 281)
- Актуальность обозначений производителей
- Наличие source для каждой записи

## Примеры использования

### Поиск аналога ГОСТ → SKF

```bash
grep "^180305," kb/ru/bearings/datasets/nomenclature.csv | grep SKF
# Результат: 180305,6305,SKF,...
```

### Получение характеристик подшипника

```bash
grep "^6305," kb/ru/bearings/datasets/catalog.csv
# Результат: 6305,ball_deep_groove,25,62,17,22.5,11.2,10000,0.13
```

### Список производителей из России

```bash
grep "Russia" kb/ru/bearings/datasets/manufacturers.csv
```

## Частые ошибки

- **Дублирование записей**: всегда проверяйте уникальность перед добавлением
- **Отсутствие source_file**: каждая запись должна иметь трассировку
- **Устаревшие обозначения**: проверяйте актуальность по последним каталогам
- **Несоответствие ISO**: габариты и грузоподъёмность должны соответствовать стандартам
